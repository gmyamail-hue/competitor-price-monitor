# 竞品监控 · 部署与配置说明（给非技术同学）

实现功能：

- 每天 **北京时间早上 5:00** 自动发邮件
- 附件为 **Excel 可打开的 CSV 报表**
- 网页上可随时 **暂停 / 恢复**（退出机制）

---

|------|------|----------|
| GitHub | 放代码 | https://github.com |
| Vercel | 托管网站 + 定时任务 | https://vercel.com |

## 二、把代码推到 GitHub

确保仓库里有这些文件（不要漏 `package.json`、`api` 文件夹）：

- `index.html`、`style.css`、`script.js`、`monitor.js`
- `api/`、`lib/`、`vercel.json`、`package.json`

用网页上传或 Git 推送均可。

---

## 三、在 Vercel 部署

1. 登录 https://vercel.com ，用 GitHub 登录  
2. **Add New → Project** → 选择 `competitor-price-monitor`  
3. 点击 **Deploy**（首次会失败也没关系，先完成第四步环境变量再 Redeploy）

---

## 四、添加 Upstash Redis（存网址 + 暂停状态）

1. 进入 Vercel 项目 → **Storage** → **Create Database**  
2. 选择 **Upstash Redis** → 创建  
3. 连接到当前项目（Connect to Project）  
4. 会自动添加 `UPSTASH_REDIS_REST_URL` 和 `UPSTASH_REDIS_REST_TOKEN`

---

## 五、配置 Resend 发邮件

1. 注册 https://resend.com ，进入 **API Keys** 创建密钥  
2. 在 Vercel 项目 → **Settings → Environment Variables** 添加：

| 变量名 | 填什么 |
|--------|--------|
| `RESEND_API_KEY` | Resend 的 API Key |
| `MAIL_FROM` | 发件人。测试可填 `onboarding@resend.dev` |
| `MAIL_TO` | **你的收件邮箱** |
| `ADMIN_PIN` | **你自己设的管理密码**（网页上暂停/添加网址要用） |
| `CRON_SECRET` | 随便一串长字符，如 `mySecretCron2026xyz` |

3. 每个变量勾选 **Production**（也可勾选 Preview）

---

## 六、重新部署

环境变量保存后：

**Deployments → 最新一条右侧 ⋮ → Redeploy**

---

## 七、使用说明

1. 打开 Vercel 给你的网址（如 `https://xxx.vercel.app`）  
2. 找到 **「自动邮件监控」** 区域  
3. 输入 **管理密码**（即 `ADMIN_PIN`）  
4. 添加要监控的竞品网址  
5. 点 **「立即发送测试邮件」** 确认能收到  
6. 需要停止时：点 **「暂停每日邮件」**  
7. 需要恢复时：点 **「恢复每日邮件」**

---

## 八、定时说明

- 配置在 `vercel.json`：`0 21 * * *`（UTC）= **北京时间 05:00**
- 暂停后，定时任务仍会触发，但 **不会发邮件**

---

## 九、常见问题

**Q：收不到邮件？**  
- 检查 `MAIL_TO` 是否正确  
- 在 Resend 控制台看是否发送失败  
- 先点网页上的「测试邮件」

**Q：价格是「失败」？**  
- 部分网站禁止自动访问，属正常  
- 可继续用页面上方「粘贴源码」手动解析

**Q：本地双击 index.html 没有自动邮件功能？**  
- 自动邮件必须部署在 Vercel 后才生效
